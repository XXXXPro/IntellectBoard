Текстовые данные

Таблица prefix_text -- универсальное хранилище любых текстовых данных. 
Для выборки данных используется ключ из двух частей: 
oid -- номер объекта (форум, тема, пользователь, сообщение)
type -- тип хранимых данных
Для общефорумных данных (например, правил форума) используется oid=0 

В частности, сейчас предусмотрены следующие типы данных:
0 -- правила форума или раздела (для форума oid=0, для раздела -- id раздела)
1 -- вступительное слово (oid -- аналогично правилам)
2 -- объявление форума или раздела
3 -- сериализованные данные о разделе (дополнительные настройки, кеш информации для главной страницы, и т.п.)
4 -- список стоп-последовательностей, наличие которых в сообщении делает невозможным его отправку (в основном, предназначен для запрета ссылок на недружественные сайты). 
При этом последовательность может находиться и внутри слова. 
5 -- список запрещенных слов, для которых производится замена на их более цензурные версии при отправке сообщения. 
В отличие от списка из пункта 4, последовательности из него применяются только к целым словам.
Список задается в виде запрещенное_слово=слово_для_замены, в запрещенном слове можно также использовать символы ? и * для обозначения одной буквы и произвольной последовательности

16 -- текст сообщения на форуме (oid -- идентификатор сообщения)
17 -- модераторский комментарий (oid -- идентификатор сообщения)
18 (возможно) -- упакованный список приложенных к сообщению файлов
19 -- расширенные данные о ссылках для тега blocklink

32 -- краткая информация о себе пользователя (oid -- идентификатор пользователя)
33 -- описание роли пользователя в форумной команде (пользователь должен состоять в группах со статусом "команда форума")

48 -- сериалиованные данные о теме (аналогично 3)
